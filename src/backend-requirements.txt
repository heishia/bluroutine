# Bluroutine Backend 최소 구현 요구사항

## 1. 사용자 인증 API
### 1.1 소셜 로그인
- [ ] POST /auth/kakao - 카카오 로그인
- [ ] POST /auth/google - 구글 로그인  
- [ ] POST /auth/naver - 네이버 로그인
- [ ] POST /auth/logout - 로그아웃

### 1.2 이메일 회원가입/로그인
- [ ] POST /auth/signup - 이메일 회원가입
- [ ] POST /auth/login - 이메일 로그인

### 1.3 사용자 정보
- [ ] GET /auth/me - 현재 사용자 정보 조회
- [ ] JWT 토큰 기반 인증 구현

## 2. 루틴 관리 API
### 2.1 루틴 CRUD
- [ ] GET /routines - 사용자 루틴 목록 조회
- [ ] POST /routines - 루틴 추가
- [ ] PUT /routines/:id - 루틴 수정
- [ ] DELETE /routines/:id - 루틴 삭제
- [ ] PUT /routines/reorder - 루틴 순서 변경

### 2.2 루틴 완료 상태 관리
- [ ] GET /routine-progress?date=YYYY-MM-DD - 특정 날짜 루틴 완료 상태 조회
- [ ] POST /routine-progress - 루틴 완료 상태 토글
- [ ] GET /routine-progress/week?startDate=YYYY-MM-DD - 주간 완료 상태 조회

## 3. 데이/세트 관리 API
### 3.1 데이 세션 관리
- [ ] GET /day-records/:date - 특정 날짜 세션 데이터 조회
- [ ] POST /day-records - 데이 세션 저장/업데이트
- [ ] PUT /day-records/:date - 데이 세션 전체 업데이트

### 3.2 사용자 활동 관리
- [ ] GET /activities - 사용자 활동 목록 조회
- [ ] POST /activities - 활동 추가
- [ ] PUT /activities/:id - 활동 수정
- [ ] DELETE /activities/:id - 활동 삭제
- [ ] PUT /activities/reorder - 활동 순서 변경

## 4. 통계 API (선택사항)
### 4.1 기본 통계만 구현
- [ ] GET /stats/routine-summary?year=2025&month=9 - 월별 루틴 완료율
- [ ] GET /stats/weekly-progress - 주간 진행률 데이터

## 5. 데이터베이스 스키마
### 5.1 필수 테이블
- [ ] users (id, provider, name, email, created_at, updated_at)
- [ ] routines (id, user_id, time_action, routine_text, emoji, order_index, created_at, updated_at)
- [ ] routine_progress (id, user_id, routine_id, date, is_completed, created_at, updated_at)
- [ ] day_records (id, user_id, date, sessions_json, created_at, updated_at)
- [ ] user_activities (id, user_id, name, color, order_index, created_at, updated_at)

## 6. 기술 스택 권장사항
- [ ] Node.js + Express.js 또는 Python + FastAPI
- [ ] PostgreSQL 또는 MySQL
- [ ] JWT 토큰 인증
- [ ] 소셜 로그인 SDK 연동

## 7. 배포 최소 요구사항
- [ ] HTTPS 지원
- [ ] CORS 설정 (프론트엔드 도메인 허용)
- [ ] 환경변수 관리 (API 키, DB 연결 정보)

## 주의사항
- 모든 API는 사용자별 데이터 격리 필수
- 소셜 로그인 실패 시 적절한 에러 핸들링
- 루틴/활동 순서 변경 시 트랜잭션 처리
- sessions_json은 DaySession[] 배열을 JSON으로 저장
